# MMR1 项目理解路线图

## 项目一句话概述
用强化学习（RL）训练多模态视觉语言模型（如 Qwen2.5-VL），核心创新是 VAS（Variance-Aware Sampling）——智能挑选训练样本，让梯度信号更稳定。

### 第三步：奖励函数
**文件**：`verl/workers/reward/custom.py`

- RL 的核心：定义什么是"答对"
- 支持多种评分方式：`math`、`r1v`、`openr1`、`openr1_wo_LMM`
- 逻辑相对独立，容易单独理解

**实际使用的 reward 公式**（`openr1_wo_LMM`，cosine len 全设为 0 故省略）：
```
overall = accuracy + format + repetition_penalty

accuracy：多层验证（<answer>标签 → \boxed{} → normalize → progressive_verify → math_verify）
          返回 1.0 / 0.5（全文匹配） / 0.0
format：  <think>...</think><answer>...</answer> 各出现一次 → 1.0，否则 0.0
repetition_penalty：40-gram 重复率 × (−0.5)，范围 [−0.5, 0]
```
值域：正确+格式对 = 2.0，错误+格式错+严重重复 = −0.5。

### 第四步：MMR1 核心创新 VAS
**文件**：先看 `verl/curriculum/metrics.py`，再看 `verl/curriculum/samplers.py`

**论文定义的 VPS 公式**：
```
VPS(x) = α × OVS(x) + β × TDS(x)

OVS(x) = P(x) × (1 - P(x))          ← Bernoulli 方差，即 learnability
  P(x) = 每道题 N 次 rollout 的通过率
  → P=0.5 时 OVS 最大；全对/全错时 OVS=0

TDS(x) = Diversity({y_i})             ← 论文未固定，给了两个候选
  候选1：inverse self-BLEU（BLEU 越低 = 多样性越高）
  候选2：distinct-n（唯一 n-gram 占比，越高越多样）
```

**实际配置（mmr1.yaml）只用了 OVS，TDS 未启用**：
```
VPS(x) = 1.0 × OVS(x)    （β = 0）
```
代码里 self-BLEU、edit-distance、distinct-3 均已实现，但配置中未开启。

**如需启用 TDS，修改 yaml**：
```yaml
curriculum_metrics:
  - learnability
  - distinct_3          # TDS 候选1，正权重
  - edit_distance       # TDS 候选2，正权重
  - self_bleu_123       # TDS 候选3，必须用负权重（越高=越雷同）
curriculum_metric_weights:
  - 1.0
  - 0.3
  - 0.2
  - -0.2                # 负数才是正确方向
```

**代码比论文多的两个工程处理**（`_compute_curriculum_weights`）：
```
1. Min-Max 归一化：把权重压到 [0,1]，防止极端值主导采样
2. EMA 平滑：new = momentum × old + (1-momentum) × computed
             防止权重每次更新时剧烈跳变（论文直接覆盖更新）
```

**混合采样策略**（samplers.py `MixedCurriculumSampler`）：
```
每个 batch = λ × VPS加权样本（高价值）
           + (1-λ) × 随机样本（防止遗忘）
实际配置：λ = 0.5
```

#### samplers.py 核心源码笔记

三个类的定位：
```
CurriculumWeightedSampler   → 纯加权采样，实际未用到
MixedCurriculumSampler      → 加权+随机混合，训练阶段使用
PaddedSequentialSampler     → 顺序+补齐，计算 VPS 时遍历全数据集用
```

**MixedCurriculumSampler 核心设计**

初始化时维护两套独立的索引池：
```python
# 加权池：按 VPS 权重采样，高分题被选中概率更大
self.weighted_sampler = WeightedRandomSampler(weights, num_samples=dataset_size)

# 随机池：提前 shuffle 一遍全数据集存起来，按顺序取，取完从头循环
self._init_random_indices()   # → self.random_indices_pool
```

`__iter__` 生成 batch 索引序列：
```python
for batch_idx in range(num_batches):
    batch_weighted = weighted_indices[w_start : w_start + n_weighted]  # 从加权池切
    batch_random = self._get_next_random_indices(n_random)              # 从随机池顺序取
    # 交错合并：[w0, r0, w1, r1, ...]
```

交错排列在这个项目里**没有实质意义**——训练时 batch 会被二次切分成 mini-batch / micro-batch（`dp_actor.py`），连续切分会破坏交错顺序，梯度计算层面感知不到。真正起作用的只是 `mixture_ratio` 保证的比例。

**断点续训的关键：consumed_batches**
```python
# 权重更新后重建索引时，随机池从上次停下的位置继续
self.random_indices_position = self.get_training_random_position()
# = (consumed_batches × n_random_per_batch) % pool_size
```
保证恢复训练后随机顺序和未中断时一致，训练可复现。

**update_weights 更新逻辑**：
```python
def update_weights(self, new_weights):
    self.weighted_sampler = WeightedRandomSampler(new_weights, ...)
    self.cached_mixed_indices = None   # 清缓存，强制下次 __iter__ 重算
    # 随机池不重置，保持连续性
```

支持完整 `state_dict` / `load_state_dict` 检查点。

#### metrics.py 核心源码笔记

**1. learnability（OVS 实现）**

```python
pass_rate = acc_reward_tensor.mean(dim=-1).mean(dim=-1)
learnability = pass_rate * (1 - pass_rate)
```

输入 `acc_reward_tensor` shape：`(batch_size, rollout_n, seq_len)`。
先对 seq_len 取均值得到每条 rollout 的得分，再对 rollout_n 取均值得到 pass rate，最后算伯努利方差。

**2. self-BLEU 系列**

```python
# calculate_self_bleu_n：只看第 n 阶 n-gram
weights = tuple([1.0] if n == 1 else [0.0] * (n-1) + [1.0])

# calculate_self_bleu_123：1/2/3-gram 均匀混合（实际 TDS 用这个）
weights = (1/3, 1/3, 1/3)

# 核心循环：每条序列轮流当 hypothesis，其余当 references
for i, hyp in enumerate(tokenized_sequences):
    refs = tokenized_sequences[:i] + tokenized_sequences[i+1:]
    score = sentence_bleu(refs, hyp, weights=weights,
                          smoothing_function=SmoothingFunction().method1)
```

- 分数越高 = 各条回答越雷同 = 多样性越差
- 用作 TDS 时**必须设负权重**，代码本身不取反
- `SmoothingFunction().method1`：防止 n-gram 精确率为 0 时 log(0) 崩塌

**3. Ray 分布式计算模式**

metrics.py 里所有计算密集型函数都用同一套模式：

```python
# Step1：数据只序列化一次，放共享内存
shared_data_ref = ray.put(tokenized_sequences)   # O(1) 传输

# Step2：每道题一个任务，按题切片
task_params = [(shared_data_ref, i*N, (i+1)*N) for i in range(batch_size)]

# Step3：分块提交，避免调度器过载
#   self-BLEU chunk=8（O(N²) 计算，更重）
#   distinct-n / edit-distance chunk=100
for chunk in chunks:
    futures = [task.remote(...) for ... in chunk]

# Step4：统一收集
scores = ray.get(all_futures)   # 阻塞等所有任务完成
return torch.tensor(scores)
```

**为什么用 `ray.put`**：不用的话每个 remote 函数收到的是数据的独立副本，序列化 batch_size 次，内存开销 O(N²)；用 `ray.put` 只序列化一次，所有 worker 共享引用，O(N)。

**learnability 的特殊之处**：它不走"每题一个 worker"的模式，而是整个 batch 一个 remote 任务，因为它需要跑 reward_fn 打分，必须把生成结果和题目元信息合并后才能计算：
```python
batch = batch.repeat(repeat_times=rollout_n, interleave=True)  # 每题复制N份交错
batch = batch.union(gen_batch_output)   # 合并生成结果
# 再调 reward_fn → 得到 0/1 → 算 pass_rate → 算 P(1-P)
```

### 第五步：配置文件
**文件**：`examples/mmr1.yaml`

看完前四步再回来看配置，所有参数都能对上：
- `data.sampling_strategy: curriculum` — 开启 VAS
- `data.curriculum_metrics: [learnability]` — 只用 OVS，TDS 未启用
- `data.curriculum_metric_weights: [1.0]` — 单指标权重
- `data.curriculum_mixture_ratio: 0.5` — λ=0.5，一半加权一半随机
- `data.curriculum_rollout_n: 8` — 每题生成 8 条 rollout 算 pass rate
- `data.curriculum_update_freq: 4` — 每 4 步重算一次 VPS
- `data.curriculum_momentum: 0.0` — EMA 动量（0.0 = 无EMA，直接覆盖；公式 `new = momentum×old + (1-momentum)×computed`）
- `algorithm.adv_estimator: grpo` — 使用 GRPO 算法
