# MMR1 项目理解路线图

## 项目一句话概述
用强化学习（RL）训练多模态视觉语言模型（如 Qwen2.5-VL），核心创新是 VAS（Variance-Aware Sampling）——智能挑选训练样本，让梯度信号更稳定。

---

## 整体架构图
```
训练入口 verl/trainer/main.py
      │
      ▼
RayPPOTrainer.fit()          ← 主控制器
      │
      ├── MixedCurriculumSampler   ← VAS 采样（MMR1 创新）
      │       50% 高VPS样本 + 50% 随机
      │
      ├── Actor + vLLM Rollout     ← 生成回答
      │
      ├── Reward Function          ← 打分
      │
      └── GRPO 更新参数            ← 强化学习
```

### 第三步：奖励函数
**文件**：`verl/workers/reward/custom.py`

- RL 的核心：定义什么是"答对"
- 支持多种评分方式：`math`、`r1v`、`openr1`、`openr1_wo_LMM`
- 逻辑相对独立，容易单独理解

**实际使用的 reward 公式**（`openr1_wo_LMM`，cosine len 全设为 0 故省略）：
```
overall = accuracy + format + repetition_penalty

accuracy：多层验证（<answer>标签 → \boxed{} → normalize → progressive_verify → math_verify）
          返回 1.0 / 0.5（格式不规范但内容对） / 0.1（选择题猜对兜底） / 0.0
format：  <think>...</think><answer>...</answer> 各出现一次 → 1.0，否则 0.0
repetition_penalty：40-gram 重复率 × (−0.5)，范围 [−0.5, 0]
```
值域：正确+格式对 = 2.0，错误+格式错+严重重复 = −0.5。

### 第四步：MMR1 核心创新 VAS
**文件**：先看 `verl/curriculum/metrics.py`，再看 `verl/curriculum/samplers.py`

**论文定义的 VPS 公式**：
```
VPS(x) = α × OVS(x) + β × TDS(x)

OVS(x) = P(x) × (1 - P(x))          ← Bernoulli 方差，即 learnability
  P(x) = 每道题 N 次 rollout 的通过率
  → P=0.5 时 OVS 最大；全对/全错时 OVS=0

TDS(x) = Diversity({y_i})             ← 论文未固定，给了两个候选
  候选1：inverse self-BLEU（BLEU 越低 = 多样性越高）
  候选2：distinct-n（唯一 n-gram 占比，越高越多样）
```

**实际配置（mmr1.yaml）只用了 OVS，TDS 未启用**：
```
VPS(x) = 1.0 × OVS(x)    （β = 0）
```
代码里 self-BLEU、edit-distance、distinct-3 均已实现，但配置中未开启。

**如需启用 TDS，修改 yaml**：
```yaml
curriculum_metrics:
  - learnability
  - distinct_3          # TDS 候选1，正权重
  - edit_distance       # TDS 候选2，正权重
  - self_bleu_123       # TDS 候选3，必须用负权重（越高=越雷同）
curriculum_metric_weights:
  - 1.0
  - 0.3
  - 0.2
  - -0.2                # 负数才是正确方向
```

**代码比论文多的两个工程处理**（`_compute_curriculum_weights`）：
```
1. Min-Max 归一化：把权重压到 [0,1]，防止极端值主导采样
2. EMA 平滑：new = momentum × old + (1-momentum) × computed
             防止权重每次更新时剧烈跳变（论文直接覆盖更新）
```

**混合采样策略**（samplers.py `MixedCurriculumSampler`）：
```
每个 batch = λ × VPS加权样本（高价值）
           + (1-λ) × 随机样本（防止遗忘）
实际配置：λ = 0.5
```

#### samplers.py 核心源码笔记

三个类的定位：
```
CurriculumWeightedSampler   → 纯加权采样，实际未用到
MixedCurriculumSampler      → 加权+随机混合，训练阶段使用
PaddedSequentialSampler     → 顺序+补齐，计算 VPS 时遍历全数据集用
```

**MixedCurriculumSampler 核心设计**

初始化时维护两套独立的索引池：
```python
# 加权池：按 VPS 权重采样，高分题被选中概率更大
self.weighted_sampler = WeightedRandomSampler(weights, num_samples=dataset_size)

# 随机池：提前 shuffle 一遍全数据集存起来，按顺序取，取完从头循环
self._init_random_indices()   # → self.random_indices_pool
```

`__iter__` 生成 batch 索引序列：
```python
for batch_idx in range(num_batches):
    batch_weighted = weighted_indices[w_start : w_start + n_weighted]  # 从加权池切
    batch_random = self._get_next_random_indices(n_random)              # 从随机池顺序取
    # 交错合并：[w0, r0, w1, r1, ...]
```

交错排列在这个项目里**没有实质意义**——训练时 batch 会被二次切分成 mini-batch / micro-batch（`dp_actor.py`），连续切分会破坏交错顺序，梯度计算层面感知不到。真正起作用的只是 `mixture_ratio` 保证的比例。

**断点续训的关键：consumed_batches**
```python
# 权重更新后重建索引时，随机池从上次停下的位置继续
self.random_indices_position = self.get_training_random_position()
# = (consumed_batches × n_random_per_batch) % pool_size
```
保证恢复训练后随机顺序和未中断时一致，训练可复现。

**update_weights 更新逻辑**：
```python
def update_weights(self, new_weights):
    self.weighted_sampler = WeightedRandomSampler(new_weights, ...)
    self.cached_mixed_indices = None   # 清缓存，强制下次 __iter__ 重算
    # 随机池不重置，保持连续性
```

支持完整 `state_dict` / `load_state_dict` 检查点。

#### metrics.py 核心源码笔记

metrics.py 整体分三层：

**层 1 — 纯计算函数**（本地执行，无 Ray）
- `calculate_distinct_n` / `calculate_self_bleu_n` / `calculate_self_bleu_123` / `calculate_pairwise_edit_distance`

**层 2 — Ray remote 单任务**（每题一个 worker）
- `calculate_single_bleu_score` / `calculate_single_edit_distance` / `calculate_single_distinct_n`
- 入参是 `shared_data_ref`（ObjectRef），不是数据本身，由 `ray.put` 预先放入共享内存

**层 3 — Ray remote 批量入口**（trainer 实际调用的）
- `calculate_self_bleu_metric` / `calculate_edit_distance_metric` / `calculate_distinct_n_metric`
  → 走统一四步模式（ray.put → 切任务参数 → 分块提交 → ray.get 收集）
- `calculate_learnability_metric_with_batch_data`
  → 特殊：整个 batch 打包成一个 remote task（需要运行 reward_fn，无法 per-item 并行）

---

**1. learnability（OVS 实现）**

```python
pass_rate = acc_reward_tensor.mean(dim=-1).mean(dim=-1)
learnability = pass_rate * (1 - pass_rate)
```

输入 `acc_reward_tensor` shape：`(batch_size, rollout_n, seq_len)`。
先对 seq_len 取均值得到每条 rollout 的得分，再对 rollout_n 取均值得到 pass rate，最后算伯努利方差。

更精确地说：`reward_metrics["accuracy"]` 是长度 `batch_size × rollout_n` 的标量列表（每条 rollout 一个 0/1 分），`.view(batch_size, rollout_n, -1)` 后 -1=1，shape 实际是 `(batch_size, rollout_n, 1)`。第一次 `.mean(dim=-1)` 是 no-op（dim 大小为 1），第二次对 rollout_n 维取均值，得到每题的通过率 `(batch_size,)`。写两次是为了代码通用性（accuracy 将来可能是多 token）。

**2. self-BLEU 系列**

```python
# calculate_self_bleu_n：只看第 n 阶 n-gram
weights = tuple([1.0] if n == 1 else [0.0] * (n-1) + [1.0])

# calculate_self_bleu_123：1/2/3-gram 均匀混合（实际 TDS 用这个）
weights = (1/3, 1/3, 1/3)

# 核心循环：每条序列轮流当 hypothesis，其余当 references
for i, hyp in enumerate(tokenized_sequences):
    refs = tokenized_sequences[:i] + tokenized_sequences[i+1:]
    score = sentence_bleu(refs, hyp, weights=weights,
                          smoothing_function=SmoothingFunction().method1)
```

- 分数越高 = 各条回答越雷同 = 多样性越差
- 用作 TDS 时**必须设负权重**，代码本身不取反
- `SmoothingFunction().method1`：防止 n-gram 精确率为 0 时 log(0) 崩塌

**3. Ray 分布式计算模式**

metrics.py 里所有计算密集型函数都用同一套模式：

```python
# Step1：数据只序列化一次，放共享内存
shared_data_ref = ray.put(tokenized_sequences)   # O(1) 传输

# Step2：每道题一个任务，按题切片
task_params = [(shared_data_ref, i*N, (i+1)*N) for i in range(batch_size)]

# Step3：分块提交，避免调度器过载
#   self-BLEU chunk=8（O(N²) 计算，更重）
#   distinct-n / edit-distance chunk=100
for chunk in chunks:
    futures = [task.remote(...) for ... in chunk]

# Step4：统一收集
scores = ray.get(all_futures)   # 阻塞等所有任务完成
return torch.tensor(scores)
```

**为什么用 `ray.put`**：不用的话每个 remote 函数收到的是数据的独立副本，序列化 batch_size 次，内存开销 O(N²)；用 `ray.put` 只序列化一次，所有 worker 共享引用，O(N)。

**learnability 的特殊之处**：它不走"每题一个 worker"的模式，而是整个 batch 一个 remote 任务，因为它需要跑 reward_fn 打分，必须把生成结果和题目元信息合并后才能计算：
```python
batch = batch.repeat(repeat_times=rollout_n, interleave=True)  # 每题复制N份交错
batch = batch.union(gen_batch_output)   # 合并生成结果
# 再调 reward_fn → 得到 0/1 → 算 pass_rate → 算 P(1-P)
```

**③ `_compute_curriculum_weights`（ray_trainer.py:646）**

这是 VAS 的"体检入口"——对全部训练数据跑一遍推理 + 打分，算出每道题的 learnability，归一化后返回。分四段：

**第一段：准备工作**

```python
curriculum_weights = torch.zeros(len(self.train_dataset))  # 最终结果容器，14996 个 0

reward_fn_ref = ray.put(self.reward_fn)
# reward_fn = CustomRewardManager（workers/reward/custom.py）
# 序列化一次放共享内存，所有 remote task 共享同一引用

padded_sampler = PaddedSequentialSampler(dataset, batch_size=curriculum_rollout_batch_size)
# curriculum/samplers.py：顺序遍历，末尾补齐到 batch_size 整数倍（重复最后一条）
# 作用：保证每个 batch 大小完全一致，后续 view(batch_size, rollout_n, -1) 不出错

curriculum_dataloader = StatefulDataLoader(dataset, sampler=padded_sampler, num_workers=8)
```

**第二段：异步提交所有 batch 的计算（生产者）**

```python
metric_futures_with_batch_idx = []   # 存 (batch_idx, [ObjectRef, ...]) 列表

for batch_idx, batch_data in enumerate(curriculum_dataloader):
    futures = self._calculate_curriculum_metric_with_batch(
        batch_data=batch_data,
        reward_fn_ref=reward_fn_ref
    )
    metric_futures_with_batch_idx.append((batch_idx, futures))
    # ← 这里只提交任务，没有 ray.get()，不阻塞，立刻处理下一个 batch
```

关键设计：先把所有 batch 的任务全部提交给 Ray，让 Ray 并发执行（vllm 推理 + 打分可以跨 batch 并行），再统一收集。如果改成"提交一个等一个"，Ray 退化成串行，浪费。

**第三段：统一收集结果（消费者）**

```python
for batch_idx, futures in metric_futures_with_batch_idx:
    metric_results = ray.get(futures)   # 阻塞，等这个 batch 的 learnability 算完

    combined_metric = combine_metric_results(   # curriculum/metrics.py
        metric_results,
        weights=[1.0],        # curriculum_metric_weights
        metrics=["learnability"],
    )
    # combined_metric: shape (batch_size,)，每题的 P*(1-P)

    # 写回，处理末尾 padding
    start_idx = batch_idx * batch_size
    end_idx = min(start_idx + batch_size, len(train_dataset))
    valid_samples = end_idx - start_idx   # 去掉 padding 的部分
    curriculum_weights[start_idx:end_idx] = combined_metric[:valid_samples]
```

**第四段：Min-Max 归一化**

```python
min_w = curriculum_weights.min()
max_w = curriculum_weights.max()
curriculum_weights = (curriculum_weights - min_w) / (max_w - min_w + 1e-8)
# 把所有题的 learnability 压到 [0, 1]
# 防止极端值主导采样概率（全是 P=0.5 的难题 or 全是简单题时同样有效）
```

⚠️ 这个函数耗时 6-7 小时（14996 题 × 8 rollout ≈ 12 万次推理），快速测试时用 `torch.ones()` 跳过。

**逐环详解**

**第一环：`__init__` → `_create_dataloader()`（训练启动，执行一次）**

`ray_trainer.py:357`

训练进程启动时，`__init__` 调用 `_create_dataloader()`，做两件事：

1. **初始化权重**

   先尝试从磁盘加载上次保存的 curriculum 状态（`checkpoints/curriculum_weights/*.pt`）：
   - 有 checkpoint → 恢复 `curriculum_weights`（上次算好的每题 learnability 分数）
   - 没有 → 用 `torch.ones(15000)`，所有题权重相等（快速测试用，跳过 6h 初始化）

2. **创建 sampler 并挂入 DataLoader**

   ```python
   sampler = self.curriculum_sampler = MixedCurriculumSampler(dataset, weights, ...)
   # ↓ 关键一行
   self.train_dataloader = StatefulDataLoader(dataset, sampler=sampler, ...)
   ```

   原始 veRL 是 `shuffle=True`，MMR1 把它换成 `sampler=sampler`。DataLoader 每次迭代时调用 `sampler.__iter__()` 来决定取哪些 index，curriculum 就从这里介入数据流。

---

**第二环：`fit()` 每个 step 取 batch（`MixedCurriculumSampler.__iter__`）**

`curriculum/samplers.py`

每次训练迭代，DataLoader 向 sampler 要一个 batch 的 index。`MixedCurriculumSampler` 按以下规则返回：

```
128个 index = 64个来自 WeightedRandomSampler（高 learnability 的题概率更大）
            + 64个来自 random_pool（预先 shuffle 好的全集，按顺序取）
```

这就是"课程学习"真正生效的地方——高 learnability（P≈0.5）的题被优先送进训练，但不完全排除简单/难题（防遗忘）。

---

**第三环：rollout → reward → update（正常训练，不感知 curriculum）**

`fit()` 拿到这个 batch 之后，正常走 GRPO 流程：
```
generate_sequences()  → vllm 推理，每题生成 8 条回答
reward_fn()           → openr1 打分（accuracy + format + repetition_penalty）
compute_log_probs()   → 算 actor log prob
compute_advantage()   → GRPO 算 advantage
update_actor()        → 梯度更新
```
这些环节完全不知道 curriculum 的存在，它们只是消费 DataLoader 给的 batch。

---

**第四环：每 step 末尾 → `update_position_after_batch()`**

`curriculum/samplers.py:121`

每个 step 结束，不管有没有更新 weights：

```python
self.curriculum_sampler.update_position_after_batch()
# consumed_batches += 1
```

这让 random_pool 的指针（`random_indices_position`）正确推进，断点续训时能从同一位置继续，保证训练可复现。

---

**第五环：每 56 步 → `_update_curriculum_weights()`（重算 learnability）**

`ray_trainer.py:730`

触发条件：`global_step % curriculum_update_freq == 0`

调用 `_compute_curriculum_weights()`，这是最重的一步：

**5a. 准备**（`ray_trainer.py:646`）

```python
reward_fn_ref = ray.put(self.reward_fn)  # 序列化一次放 Ray 共享内存
padded_sampler = PaddedSequentialSampler(train_dataset, batch_size)
# 顺序遍历全部 15000 条，末尾补齐到 batch_size 整数倍
```

**5b. 异步提交所有 batch**

对每个 batch，调 `_calculate_curriculum_metric_with_batch()`：
- 用 `actor_rollout_wg.generate_sequences(n=8)` → vllm 对这个 batch 的每道题生成 8 条回答
- 立刻提交 `calculate_learnability_metric_with_batch_data.remote(...)` 异步任务，**不等结果**，直接处理下一个 batch

这样所有 batch 的推理任务并发提交给 Ray，不串行等待。

**5c. 统一收集**（`curriculum/metrics.py:134`）

逐 batch `ray.get(future)` 拿结果：
```python
pass_rate = accuracy_rewards.mean(dim=-1)   # 8条回答的平均通过率
learnability = pass_rate * (1 - pass_rate)  # P*(1-P)，P=0.5 时最大=0.25
```

**5d. Min-Max 归一化**

```python
weights = (raw - min) / (max - min + 1e-8)  # 压到 [0,1]
```

**5e. EMA 平滑**（`ray_trainer.py:742`）

```python
new = 0.0 * old + 1.0 * computed   # momentum=0 → 直接覆盖
```

**5f. 保存到磁盘**

```python
# checkpoints/curriculum_weights/weights_step_N.pt
```

---

**第六环：更新 sampler + 刷新迭代器**

`ray_trainer.py:1415`

```python
self.curriculum_sampler.update_weights(new_weights)
# → 内部重建 WeightedRandomSampler，清空 cached_mixed_indices

self.dataloader_iterator = iter(self.train_dataloader)
# 必须刷新！否则迭代器还持有旧 sampler 生成的索引序列
```

从下一个 step 开始，DataLoader 就按新的 learnability 权重采样——循环回到第二环。

---

> **一句话串联**：初始化时用均匀权重启动 → 每个 step 按当前权重优先取"学习边界"的题训练 → 每 56 步对全数据集重跑推理算出新的 learnability → 归一化+平滑后更新采样权重 → 下一轮 DataLoader 按新权重取题 → 如此循环
